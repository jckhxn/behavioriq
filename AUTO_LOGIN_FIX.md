# Auto-Login Fix - Session Token Approach

**Issue**: After the previous fix, users were created successfully in both Supabase Auth and the database, but the auto-login still failed. Users were redirected to the landing page with 401 errors.

## Root Cause

The previous approach used a Supabase magic link (`generateLink` with type `magiclink`):

```typescript
window.location.href = sessionData.authLink; // Redirects to link
```

**Problems**:

1. Magic links redirect the user to the root page `/` after authentication
2. No control over where the user ends up
3. User lands on `/` without staying on payment success page
4. Session is created but user loses context

## The Fix

### New Approach: Direct Session Token Method

Instead of redirecting to a magic link, we now:

1. Generate a **recovery link** (contains access & refresh tokens in URL)
2. Extract the tokens from the URL
3. Return the tokens to the client
4. Client uses `supabase.auth.setSession()` to create session directly

### 1. Updated API Endpoint

**File**: `app/api/auth/session-from-token/route.ts`

**Key Changes**:

```typescript
// Generate recovery link (contains tokens in URL)
const { data: sessionData } = await supabaseAdmin.auth.admin.generateLink({
  type: "recovery", // Changed from 'magiclink'
  email: user.email,
  options: {
    redirectTo: `${process.env.NEXT_PUBLIC_SITE_URL}/dashboard`,
  },
});

// Extract tokens from the URL parameters
const linkUrl = new URL(sessionData.properties.action_link);
const accessToken = linkUrl.searchParams.get("access_token");
const refreshToken = linkUrl.searchParams.get("refresh_token");

// Return tokens (not the link)
return NextResponse.json({
  success: true,
  user,
  session: {
    access_token: accessToken,
    refresh_token: refreshToken,
  },
});
```

### 2. Updated Payment Success Page

**File**: `app/payment-success/page.tsx`

**Key Changes**:

```typescript
// Get session tokens from API
const sessionData = await sessionResponse.json();

if (sessionResponse.ok && sessionData.success && sessionData.session) {
  // Create Supabase client
  const { createClient } = await import("@/lib/supabase/client");
  const supabase = createClient();

  // Set session directly (no redirect!)
  const { error } = await supabase.auth.setSession({
    access_token: sessionData.session.access_token,
    refresh_token: sessionData.session.refresh_token,
  });

  if (!error) {
    console.log("✅ Supabase session created successfully!");
    return true;
  }
}
```

## How It Works Now

### Complete Flow:

1. **User completes payment** → Stripe redirects to `/payment-success?session_id=...`

2. **Payment success page loads**:
   - Shows "Setting up your account..." loading screen
   - Calls `/api/auth/generate-login-token` with session_id

3. **Generate login token**:
   - Validates Stripe session
   - Returns login token + user email

4. **Create Supabase session**:
   - Calls `/api/auth/session-from-token` with login token
   - Server generates recovery link with tokens
   - Extracts `access_token` and `refresh_token` from URL
   - Returns tokens to client

5. **Set session on client**:
   - Client calls `supabase.auth.setSession(tokens)`
   - Supabase session created in browser
   - User stays on payment success page
   - No redirect, no context loss!

6. **User is logged in**:
   - Can access dashboard
   - API calls work (no more 401 errors)
   - Can click "Start Full Report" button
